# Крестики-нолики (Tic-Tac-Toe)

Интерактивная веб-игра, разработанная в рамках лабораторной работы №23. Проект демонстрирует работу с DOM-элементами, обработку событий и основные алгоритмы на JavaScript.

## 1. Описание игры

### Правила игры
- Игровое поле представляет собой сетку 3×3.
- Первый игрок ходит крестиками (X), второй — ноликами (O).
- Игроки по очереди ставят свои символы в свободные ячейки.
- Цель игры — первым выстроить три своих символа в ряд: по горизонтали, вертикали или диагонали.

### Логика победы/поражения
- **Победа**: игрок, первым собравший три своих символа в одну линию.
- **Ничья**: если все 9 ячеек заполнены, но ни один из игроков не выстроил выигрышную комбинацию.
- После завершения игры (победа или ничья) дальнейшие клики по клеткам блокируются до перезапуска.

### Особенности реализации
- Полностью интерактивный интерфейс без перезагрузки страницы.
- Визуальная обратная связь при наведении на ячейки.
- Автоматическое определение победителя и ничьей.
- Кнопка перезапуска для начала новой игры без обновления страницы.

## 2. Игровая механика

### Как определяется победитель?
При каждом ходе вызывается функция `checkWinner()`, которая:
1. Перебирает все 8 выигрышных комбинаций (заранее определённых массивом индексов).
2. Для каждой комбинации проверяет, что все три ячейки содержат одинаковый символ и не являются пустыми.
3. При обнаружении совпадения игра останавливается, и отображается сообщение о победителе.

### Как обрабатываются клики?
1. Каждой ячейке назначается обработчик события `click`.
2. При клике проверяется:
   - активна ли игра (`gameActive === true`)
   - пуста ли ячейка
3. Если условия соблюдены:
   - символ текущего игрока записывается в массив `board`
   - обновляется отображение в DOM
   - вызывается проверка победителя или ничьей
   - если игра продолжается, ход переходит к другому игроку

### Как работает перезапуск?
Функция `restartGame()`:
- Сбрасывает `board` в исходное состояние (9 пустых строк).
- Устанавливает `currentPlayer = "X"` и `gameActive = true`.
- Обновляет текст статуса.
- Очищает все ячейки от символов X и O.

## 4. Алгоритмы

### Алгоритм проверки победы

```
Для каждой выигрышной комбинации:
    Получить значения из board по трём индексам
    Если все три значения равны и не являются пробелом:
        Игрок победил → gameActive = false
        Вывести сообщение о победе
        Выйти из функции
```


### Алгоритм определения ничьей

```
Если победитель не найден:
    Проверить, содержит ли board хотя бы один пробел (" ")
    Если пробелов нет:
        gameActive = false
        Вывести сообщение "Ничья!"
```
