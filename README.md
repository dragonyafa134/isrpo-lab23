## 1. Описание проекта

Этот проект представляет собой набор интерактивных веб-приложений, разработанных в рамках лабораторной работы №22. Основная цель — изучение основ JavaScript, взаимодействия с DOM, а также принципов рефакторинга и модульной архитектуры.

На данный момент в проекте реализован **функциональный калькулятор** с улучшенной логикой валидации и безопасными вычислениями. Весь код организован с использованием ES6-модулей, что обеспечивает его переиспользуемость и простоту поддержки.

## 2. Функциональность

### Поддерживаемые операции
Калькулятор поддерживает базовые арифметические действия:
- **Сложение** (`+`)
- **Вычитание** (`-`)
- **Умножение** (`*`)
- **Деление** (`/`)
- Ввод **десятичных дробей** (`.`)

### Ограничения
- Калькулятор не поддерживает ввод скобок и сложных математических функций.
- При вводе выражения, содержащего некорректные символы или последовательности (например, два оператора подряд `5++3`), вычисление не будет выполнено.
- Деление на ноль обрабатывается как ошибка.

### Обработка ошибок
- **Валидация ввода:** Код предотвращает ввод некорректных последовательностей (например, `..` или `*/`).
- **Безопасное вычисление:** Вместо прямого использования `eval()` применяется предварительная проверка выражения с помощью методов класса `Calculator`. Это снижает риск выполнения потенциально опасного кода.
- **Сообщения пользователю:** В случае ошибки (деление на ноль, неверный синтаксис) на дисплее отображается сообщение **"Ошибка"**, после чего состояние калькулятора автоматически сбрасывается при следующем нажатии.

### Описание класса `Calculator`

Класс `Calculator` инкапсулирует всю логику работы калькулятора, отделяя её от управления DOM. Это повышает читаемость и тестируемость кода.

#### Основные свойства:
- `displayElement`: ссылка на DOM-элемент (поле ввода).
- `expression`: строка, хранящая текущее математическое выражение.
- `hasError`: флаг, указывающий на состояние ошибки.

#### Основные методы:
- `init()`: инициализирует калькулятор, навешивая обработчики событий на кнопки.
- `handleButtonClick(value)`: главный маршрутизатор, определяющий, что делать при нажатии (очистка, вычисление или добавление символа).
- `isValidExpression()`: валидация выражения перед вычислением.
- `safeEvaluate()`: безопасная обертка для вычисления выражения.
- `updateDisplay()`: обновляет интерфейс на основе текущего состояния.
- `showError()`: переводит калькулятор в режим ошибки.

### Схема взаимодействия компонентов

1. **Пользователь** нажимает кнопку в `index.html`.
2. **Событие `click`** обрабатывается функцией в `main.js`, которая вызывает метод `handleButtonClick` у экземпляра класса `Calculator`.
3. **Класс `Calculator`** изменяет своё внутреннее состояние (`expression` или `hasError`).
4. **Метод `updateDisplay`** обновляет значение в DOM-элементе `#display`.
5. Браузер отрисовывает обновленный интерфейс.



## 4. Логика работы
Нажатие цифры или оператора:
* Проверяется, не находится ли калькулятор в состоянии ошибки. Если да, состояние сбрасывается.
* Символ добавляется к строке expression.
* Обновляется дисплей.
* Нажатие кнопки C (Clear):
* Вызывается метод clear().
* Строка expression очищается.
* Флаг hasError сбрасывается.
* а дисплее отображается 0.

Нажатие кнопки = (Equal):
* Вызывается метод safeEvaluate().
* Выражение проходит валидацию через isValidExpression().
* Если валидация успешна, вычисляется результат.
* Если результат конечный и корректен, он отображается на экране и сохраняется как новое выражение.
* Если возникает ошибка (деление на ноль, неверный синтаксис), вызывается showError().

### Как работает валидация ввода?
Валидация встроена в метод handleButtonClick и предотвращает некорректные последовательности на уровне пользовательского ввода:

Запрет двойных операторов: Не позволяет добавить оператор (+, -, *, /), если последний символ в выражении уже является оператором.

Запрет множественных точек: Метод hasDuplicateDot() проверяет, есть ли уже точка в текущем операнде. Если да, добавление ещё одной точки блокируется.

Обработка начала выражения: Запрещает начинать выражение с оператора умножения (*) или деления (/).

### Как происходит вычисление?
Подготовка: При нажатии = вызывается метод safeEvaluate().

Валидация: Выражение проверяется с помощью isValidExpression() на наличие запрещенных последовательностей (например, ++, */). Выражение не должно заканчиваться оператором.

Вычисление: Если валидация пройдена, используется функция eval().

Важно: eval() используется только после строгой проверки, что значительно повышает безопасность по сравнению с прямой передачей пользовательского ввода.

### Обработка результата:

Если результат — число, он округляется для удобства (при необходимости) и выводится на дисплей.

Результат заменяет текущее выражение, что позволяет продолжать вычисления (например, 2+2=4, затем +3=7).
